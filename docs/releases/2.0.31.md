# Release Highlights

This release tightens the sidebar interaction model so page-note ordering stays predictable and the newest streamed Codex replies remain editable as soon as they finish.

## Highlights

- The sidebar now keeps index ordering and draft placement stable, with top-level drafts inserted in their natural position instead of jumping to the end.
- Child-note `+` actions now open the append draft directly after the clicked child entry, while the parent `+` still appends at the end of the thread.
- Manual drag reordering is now limited to page notes, which removes the more fragile thread-entry reorder path from the sidebar UI.
- Newly finished Codex replies no longer lose their edit-button click handlers after the stream handoff, so the latest reply can be edited immediately.
- `@codex` side-note replies now explicitly prefer compact ASCII diagrams and avoid Mermaid in the sidebar.

## Notes

- Existing Mermaid text already stored in notes is not rewritten automatically; the ASCII guidance applies to new side-note replies.
