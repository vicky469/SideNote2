# Release Highlights

This release improves how built-in agent replies understand and update Aside threads.

## Highlights

- Built-in `@codex` side-note requests now receive page-level context for page notes, so replies can account for the full current markdown page instead of only a nearby section.
- Aside's agent prompt now describes the built-in write workflow more explicitly, including page notes, side notes, side comments, and threaded child entries.
- Regenerating an agent reply now clears the previous reply before the new response completes, avoiding stale text while the retry is running.
- Streaming regenerated replies keep the thread footer focused on agent status instead of showing unrelated footer actions mid-stream.

## Notes

- There are no note-format or vault-data migrations in this release.
- The shipped plugin assets remain `main.js`, `manifest.json`, and `styles.css`.
