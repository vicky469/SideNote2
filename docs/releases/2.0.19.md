# Release Highlights

This release adds first-class CLI support for creating SideNote2 page notes and anchored notes in an existing markdown file, and clarifies the agent workflow so those requests default to simple in-note threads instead of separate wiki pages.

## Highlights

- Added `sidenote2 comment:create` so agents and users can create a new page note or anchored note directly in an existing note file.
- Updated the bundled SideNote2 guidance so phrases like `page note` and `anchored note` map to an in-note SideNote2 thread unless the user explicitly asks for a separate wiki page.
- Added regression coverage for page-note creation, anchored-note creation, and the legacy flat-comment rejection path.
