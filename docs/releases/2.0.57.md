# Release Highlights

This release fixes a mobile sidebar regression when opening side-note cards.

## Highlights

- Keeps mobile note-sidebar card taps inside the side-note panel instead of navigating the workspace into a blank-looking editor state.
- Ignores null `file-open` events from Obsidian, preserving the current note context instead of clearing the sidebar.
- Removes hidden footer action rows from layout until a card is active, avoiding empty white strips on short child comments.

## Notes

- Desktop card-click source reveal behavior is unchanged.
